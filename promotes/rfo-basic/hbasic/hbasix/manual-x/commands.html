GM.OPEN

Open the game engine

GM.OPEN {vWidth_nexp, vHeight_nexp}{,Color_sexp>} {,<Flags_nexp>} {,<Orientation_nexp>}

(all parameters are optional except where specified)

where
vWidth, vHeight set the viewport width and height.     (If not omitted, both must be present.

if omitted, they default to the device's drawable screen size.

This is a convenience for developer's who use fixed coordinates according to their development size.
Color
 color string e.g "black" or "#FF000000" for the background.

The default background color is white.
Flags
 This is a bit position flag. Only bit 5 is used.

A (default) value of 0 will fit the viewport inside the device's screen keeping the same ratio.

A value of 16 will stretch the viewport inside the device's screen potentially distorting the ratio.

This setting * will * change in the future. The first 4 bits are undefined for now, bit 5 will be reserved for this feature.
Orientation
 0 = landscape (default)
1 = portrait
-1 = change with sensors

e.g
GM.OPEN
Opens the game using all the defaults.

The developer will have to compensate for different screens in his own code.
GM.OPEN 480,800
Opens the game using a viewport of 480x800.

The engine will map this size to the device's screen, * fitting * the view inside the screen.
GM.OPEN "LightBlue",,1
Opens the game using a background color in protrait mode using the device's screen size.


About Viewports

A viewport is a map of the screen onto a real device.
Typically the developer uses coodinates of the viewport when coding.
The viewport is set with GM.OPEN. The runtime will try to fit the viewport to the real size.

If you don't use a viewport, the coordinates are mapped one-to-one to the real device, whatever that device may be. This may give strange results for different sized devices. In this case, it is up to the developer to compensate his coordinates by using GM.SCREEN to get the real size.

Coordinate System

libGDX is the internal game library and uses by default a screen coordinate system where
0,0 is the bottom left hand corner.
GM.ANI.ADD

Add Animation

GM.ANI.ADD     A_index_nvar, image_path_sexp {,frames_nexp, fW_nexp, fH_nexp}

A_index and image_path are required.

where
A_index
 The variable to get the new animation index.
image_path
 The file path of the image.

The path is relative to the rfo-basic/data directory.

This is usually a sprite sheet.
A sprite sheet consists of multple images joined together by one or more rows.

The frames are cut out according to the number of frames required and by the number of columns and rows available.

It is possible to have just 1 frame. If the optional parameters are not given,
they default to 1 frame with the fW and fH the same as the image.
frames
 The number of frames of the animation. These will be cut out of the image.
fW
 The animation frame width.

This width is cut out for each frame.
The number of columns inside the image should be image_width / fW.
fH
 The animation frame height.

This height is cut out for each frame.
The number of rows inside the image should be image_height / fH.

e.g
GM.ANI.ADD a, "sprite-sheet.png", 16, 32,33
add an animation, cutting out 16 frames from image "sprite-sheet.png" with frame size 32x33.
GM.ANI.ADD aDown,"button-down.png"
add a one frame animation using the whole image.


 Notes
This command only creates an animation. To show it, the animation needs to be attached to a sprite using GM.SPR.ANI.

When an animated sprite is rendered, the frame for display is calculated based on the last rendered time and the animation interval. Currently the interval is hard-coded to 1/20s (0.05s),
This will change in the future.
GM.SPR.ANI

Sprite - Set Animation

GM.SPR.ANI  S_index_nexp, A_index_nexp

Sets the animation for the sprite.

Both parameters are needed.

where
S_index
 The index of an existing sprite.
A_index
 The index of an existing animation.

If this index is a valid animation index, it sets the animation and turns it ON.

If this index is zero, it removes any existing animation and turns OFF sprite animation.
When an animation is removed, the sprite will go back to it's default image.

e.g
GM.SPR.ANI sPlayer, aJump
sets the sPlayer sprite to a loaded aJump animation.
GM.SPR.ANI sPlayer, 0
removes and turns off the animation for sprite sPlayer.


 Notes

GM.SPR.SIZ

Sprite - Change Size

GM.SPR.SIZ    index_nexp, {*}{width_nexp}{,{*}{height_nexp}}

At least one parameter (width or height) must be present.

where
index
 The sprite index of an existing sprite.

( A sprite index is given when you create a sprite with GM.SPR.ADD )
width
 The new width of the sprite.

If this expression is prefixed with an asterisk (*) then the value will be taken as a scale factor. e.g *2.5 means "two and a half times wider".

If there is no asterisk in front of the value then the value is in pixels.

If there is no value (omitted or illegal), then the width will not change.
height
 The new height of the sprite.

If this expression is prefixed with an asterisk (*) then the value will be taken as a scale factor. e.g *2.5 means "two and a half times taller".

If there is no asterisk in front of the value then the value is in pixels.

If there is no value (omitted or illegal), then the height will not change.

e.g
GM.SPR.SIZ 48,32
changes the size to 48 x 32 pixels, stretching or shrinking if neccessary.
GM.SPR.SIZ ,80
changes the height to 80 pixels, stretching or shrinking if neccessary.
GM.SPR.SIZ 34,*2
changes the width to 34 pixels and height to two times the current height.
